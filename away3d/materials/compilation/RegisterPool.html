class RegisterPool

package away3d.materials.compilation

@:directlyUsed

RegisterPool is used by the shader compilation process to keep track of which registers of a certain type are currently used and should not be allowed to be written to. Either entire registers can be requested and locked, or single components (x, y, z, w) of a single register. It is used by ShaderRegisterCache to track usages of individual register types.

See also:

  • away3d.materials.compilation.ShaderRegisterCache

Constructor

@:value({ persistent : true })new(regName:String, regCount:Int, persistent:Bool = true)

Creates a new RegisterPool object.

Parameters:

regName

The base name of the register type ("ft" for fragment temporaries, "vc" for vertex constants, etc)

regCount

The amount of available registers of this type.

persistent

Whether or not registers, once reserved, can be freed again. For example, temporaries are not persistent, but constants are.

Methods

addUsage(register:ShaderRegisterElement, usageCount:Int):Void

Marks a register as used, so it cannot be retrieved. The register won't be able to be used until removeUsage has been called usageCount times again.

Parameters:

register

The register to mark as used.

usageCount

The amount of usages to add.

dispose():Void

Disposes any resources used by the current RegisterPool object.

hasRegisteredRegs():Bool

Indicates whether or not any registers are in use.

removeUsage(register:ShaderRegisterElement):Void

Removes a usage from a register. When usages reach 0, the register is freed again.

Parameters:

register

The register for which to remove a usage.

requestFreeRegComponent():ShaderRegisterElement

Retrieve a single vector component that's still available.

requestFreeVectorReg():ShaderRegisterElement

Retrieve an entire vector register that's still available.
